ModADV tags

Reference of tags available in ModADV.
This module requires ModMessage/ModImage/ModSound.

If you want ModADV to process text, call the [using_mod_adv] tag.
When you call the [using_mod_adv] tag, text processing (ch/ruby/r/p/er/cm tag) is redirected to
ch_mod_adv/ruby_mod_adv/r_mod_adv/p_mod_adv/erase_message/clear_message_layer.
If you want to return text processing to the default, call the [not_using_mod_adv] tag.

If you also want to omit the [r] and [p] tags, execute the tags below.
@cr_handling !ignore

ModADV does special text interpretation.
When a message is displayed and two line feed codes appear consecutively (@cr_handling !ignore must be called), the [p] tag is executed.
If you try to display a standing picture using a character tag, a transition will be performed automatically just before the next message is displayed.
ModADV's text processing is complex, so please follow the source for details.
Basically, it would be a good idea to create a scenario in the following format.

-----
[halo normal smile]Scenario text.

[halo normal anger]The image will change.

[halo]If there is a voice, it will be played automatically.
-----

make_character(No alias)

Character operation
aliasattributeRequired?valueDescription
nonenameyesCharacter nameSpecify the character name to create or change settings.
The character name specified here will be used as the tag name.
nonemobnotrue or falseSpecifies whether the character is a mob.
If you specify true, it becomes a mob character.
Mob characters are valid characters only within the defined scenario file.
If you specify false, it will become a normal character and will be a valid character in all scenario files.
Default is false.
noneauto_voicenotrue or falseSpecify whether to automatically play character voices.
If you specify true, the voice will be played automatically.
The format specified by voice_storage_format is used for the voice file name to be played.
If you specify false, voice playback will not be performed automatically.
nonevoice_storage_formatnoformatSpecify the format of the voice file name to be automatically played.
The String#sprintf format can be used for this format.
The zero-based serial number of the voice is given as one argument.
The default is "character-name%03d".
noneimagenotrue or falseSpecify whether to have a standing image.
Specify true if you have a standing picture image.
Default is true.
nonevoicenotrue or falseSpecifies whether it has a voice.
Specify true if it has a voice.
Default is true.
nonemessagenoMessage layer numberSpecifies the message layer number used by the character.
nonevoice_buffernoVoice buffer numberSpecifies the voice buffer number used by the character.
nonehistory_voicenotrue or falseIf you specify true, the voice playback action will be added to the history.
Default is true.
nonesub_directory_namenoSub directory nameSpecify the image subdirectory name.
If the image is in /sub_directory/A/image001.png, specify sub_directory.
Specify this attribute together with part_of_directory_path.
When specifying this attribute, it is mandatory to use it together with the make tool.
nonepart_of_directory_pathnoPart of path nameSpecifies the partial path name of the image.
Specify A if the image is in /sub_directory/A/image001.png.
Specify this attribute together with sub_directory_name.
When specifying this attribute, it is mandatory to use it together with the make tool.
nonevoice_sub_directory_namenoSub directory nameSpecify the voice subdirectory name.
If the voice is located in /sub_directory/Scene001/Voice001.ogg, specify sub_directory.
The Scene001 part specifies the name of the scenario file being executed (without extension).
When specifying this attribute, it is mandatory to use it together with the make tool.
nonecolorno0xRRGGBBSpecifies the color of message text.
The default is message0's text color.
noneshadow_colorno0xRRGGBBSpecifies the shadow color of the message text.
Default is message0 shadow color.
noneedge_colorno0xRRGGBBSpecifies the edge color of the message text.
Default is message0 edge color.
nonehistory_colorno0xRRGGBBSpecifies the history text color.
The default is the history layer text color.
nonehistory_shadow_colorno0xRRGGBBSpecifies the shadow color for history text.
The default is the history layer shadow color.
nonehistory_edge_colorno0xRRGGBBSpecifies the edge color of history text.
Default is history layer edge color.
nonehistory_iconnoStorage nameSpecifies the icon to display in the history layer.
nonehistory_icon_leftnoIcon posSpecifies the left position of the icon displayed in the history layer.
nonehistory_icon_topnoIcon posSpecify the top position of the icon displayed in the history layer.
You can create characters and change settings.
Exsample:
; Change the voice file name of the character "halo" to "halo_sequential-number".
[make_character name=halo voice_storage_format=halo_%03d]

Create time zone
aliasattributeRequired?valueDescription

nonesuffixnoTime zone difference suffixSpecify the suffix of the file name to be used as the time difference.

If you use a time zone tag, the time zone difference will be applied.

[make_stage name=ent sotrage=entrance]
[make_time_zone name=ng suffix=_night]

If you specify true, children will also be included in the transition.
If you specify false, the transition will occur independently.
Default is true.
nonetimenomillisecond unitSpecify the time in milliseconds for the transition.
If omitted, 1000 milliseconds is assumed.
nonemethodno"universal" or "scroll" or otherSpecify the type of transition.
"universal" causes a universal transition to occur according to the rule, vague, time attribute.
"crossfade" performs a simple crossfade according to the time attribute.
If you specify "scroll", a scroll transition will be performed based on from and stay.
There are also types of transitions made available by plug-ins. In that case, specify the name of her transition in the method attribute, and depending on the transition, you may need to specify another attribute.
If omitted, it will be inferred from other parameters.
nonerulenoStorage nameSpecify the rule image used for universal transitions.
nonevagueno0-255Specifies a vague value to apply to the universal transition.
nonefromno"left" or "top" or "right" or "bottom"Specify the direction from which the back page image appears in the scroll transition. "left" = left, "top" = top, "right" = right, "bottom" = bottom
nonestayno"stayfore" or "stayback" or "nostay"(default)In a scroll transition, either the image on the front page stays in place and the image on the back page scrolls in from the outside ("stayfore"),
or the image on the front page scrolls out from the back, and the image on the back page scrolls out.
Specifies whether the image on the page remains in place ("stayback"), or whether the images on the front and back pages are linked and moved together ("nostay"). Specify.

Normally, use the background name tag.
